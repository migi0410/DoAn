Exploratory Data Analysis

Dataset Distribution Analysis

Visualizing the 10,000 synthetic invoices.

The Challenge: Severe Class Imbalance

The background `OTHER` label completely dominates the dataset (>1,000,000 tokens). Crucial labels like `TOTAL_COST` or `SELLER` are extremely rare. If ignored, the model will just predict `OTHER` for everything to get high accuracy.

Our Mitigations (Hướng Giải Quyết)
  • Weighted Loss Function: Assigned higher penalty weights to rare classes (e.g., TOTAL_COST) during Cross-Entropy loss calculation.
  • Evaluation Metric: Used Macro-F1 instead of Micro-F1/Accuracy to track performance equally across all classes.
Nguyen Phuoc Dai
Data Preparation

Image Preprocessing & Deskewing

Standardizing real-world photos before feeding them to the OCR engine.

Before (Raw Input)

  • Captured at arbitrary angles.
  • Complex background clutter.
  • Perspective distortion (skew).

After (Deskewed & Enhanced)

  • Crop: Document Localization (Canny Edge + Bounding Box).
  • Deskew: Hough Line Transform + Affine Rotation.
  • Noise Reduction: Gaussian Blur.
  • Enhance & Binarize: CLAHE + Adaptive Thresholding.
  • Sharpen: Unsharp Masking filter.

Why does it look gritty? The enhancement intentionally over-sharpens and boosts contrast to recover faded thermal ink. While it looks rough to the human eye, it is optimized for the OCR engine to maximize character recognition.

Post-OCR (Text Preprocessing): The Reading Order Sorting algorithm automatically rearranges OCR bounding boxes (Top-to-Bottom, Left-to-Right) to preserve structural semantic flow for the KIE model.

Experiments

Baseline Model Comparison

Comparing 3 models of equivalent size (110M - 130M parameters).

PhoBERT-base

  • Inputs: Text Only
  • Pros: Exceptional at Vietnamese language understanding.
  • Cons: Layout-blind. Cannot understand 2D spatial arrangements.

LayoutLMv1

  • Inputs: Text + 2D Bbox
  • Pros: Understands distances and structures on the page.
  • Cons: Visual-blind. Cannot see logos, font weights, or image noise.

LayoutLMv3

  • Inputs: Text + Bbox + Image Patches
  • Advantage: Uses Linear Projection (like ViT) to directly "see" the receipt image (bold ink, logos). Dramatically superior results.

Evaluation Setup

Evaluation Metrics & Fuzzy Matching

Advanced metrics to handle OCR errors and severe class imbalance in Document AI.

1. Entity-Level Metrics

Evaluates if the extracted entity strictly matches the ground truth.

  • Precision: TP / (TP + FP)
  • Recall: TP / (TP + FN)
  • F1-Score: Harmonic mean of P & R.
Macro-F1:

Averages F1 across all classes equally. Crucial for our dataset because the background "OTHER" class heavily outnumbers key entities.

2. Fuzzy Match (Binary)

Fixes the "Exact Match" penalty caused by minor OCR typos.

Levenshtein Distance Threshold:
If string similarity > 80%, it counts as 1 True Positive (Binary).
GT: "Thành tiền"
Pred: "Thành tiên" (missing accent)
---
Exact Match = 0 (False Positive)
Fuzzy Match = 90% > 80% => TP (Score = 1)

3. ANLS (Continuous)

Average Normalized Levenshtein Similarity. The gold standard metric in DocVQA.

Formula:
ANLS = 1 - (Distance / Max_Len)
(Score drops to 0 if ANLS < 0.5)
Ex 1 (Minor Typo):
GT:"150.000" vs Pred:"15.000"
Dist=1, MaxLen=7 => Score = 0.857
Ex 2 (Wrong Match):
GT:"Thuế VAT" vs Pred:"Tổng cộng"
Dist=9, MaxLen=9 => Score = 0
